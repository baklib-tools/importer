# Excel 文件使用说明

> 说明由 `preprocessing/extract-file-paths.py` 生成的拆分 Excel 各列含义与填写方式。

## 列结构

| 列 | 说明 |
|----|------|
| **打标签** | 迁入 Baklib 后用于检索、分类的标签；多个标签用 **单元格内多行**（每行一个标签），勿用空格拼接。 |
| **新目录** | 在 Baklib 中的目标逻辑路径，如 `/产品资料/图片/`。 |
| **文件名** | 不含路径与扩展名的文件名，便于人工辨认。 |
| **文件类型** | 扩展名（不含点），便于按类型筛选。 |
| **路径** | 原始完整路径，用于定位源文件。 |
| **1级、2级…** | 路径按目录拆分的层级，便于 Excel 筛选后批量填写前两列。 |

## 使用步骤（摘要）

1. 用 Excel / WPS 打开文件，对表头行启用「筛选」。
2. 按「文件类型」或某级目录列筛选目标行。
3. 批量填写「打标签」「新目录」；多标签时在单元格内 `Alt+Enter`（Windows）或 `Option+Enter`（Mac）换行。
4. 另存为新文件，保留原始表作备份。

## 新目录与标签约定

- 目录建议统一风格，如 `/一级/二级/`，避免 `\`、`:`、`*`、`?`、`"`、`<`、`>`、`|` 等非法字符。
- 多个标签必须用多行文本，避免与含空格的目录名混淆。

## 后续

填写完成后，使用 `baklib_import/import_files_to_site.py`（DAM + Page）或 `baklib_import/import_files_to_dam.py`（仅 DAM）结合 `config.json` 执行导入（路径列默认 **E 列**，与生成模板一致）。
